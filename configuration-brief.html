Configuration Brief:
AIUK Community Platform

A detailed outline of how the Discourse platform was configured for the AIUK community pilot — covering site-wide settings, group structures, category permissions, AI moderation, and onboarding flows.

About this document

This document outlines the configuration required of Discourse to create the AIUK community platform. It includes high-level wireframe descriptions which give an overview of the layout for each different type of page. This configuration is specifically designed for the pilot, and is expected to change based on real-world community feedback.

Historical record note

This document informs the Onboarding & Training: Admins & Mods section of the Amnesty Community. That training area contains the latest up-to-date information, with this document serving as a historical record.

This document is held in Google Docs and is accessible to project team members.

Terminology

  • Group — a collection of users
  • Category — a collection of topics
  • Sub-category — a smaller category within a parent category
  • Topic — a post within a category or sub-category

Site-wide elements and settings

Header and footer

Every page has a header and footer to aid navigation. The header is kept minimal to encourage exploration and guide users.

Header structure

  • Brand menu: AIUK logo (links to community home), links to AIUK website, Knowledge Hub, and About page
  • Navigation menu: Community logo, Search, Chat (Trust Level 4+), Dark Mode Toggle, User menu (Groups, Topics, My Posts, Admin, Review, Categories, Tags), Notifications

Footer structure

  • Quick links: About, Community rules (including Code of Conduct), FAQs, Privacy notice
  • Get started: Welcome & introductions, Onboarding & training, Site feedback
  • Follow us: Social media icons/links
  • Legal text: Copyright statement, charity and company details, registered office address

Email engagement

Users receive both on-platform notifications and emails about new topics, mentions, quotes, likes, and more. The default Discourse threshold of 10 minutes before switching to email notification has been kept as-is through the pilot phase.

Users can reply to topics via email — their response will appear on the platform in the relevant thread. The "Create new topics via email" feature has been deliberately disabled due to the additional cyber security work required and increased spam attack surface.

The email button colour has been changed to Amnesty dark grey (#4D4D4D) with blue link colours to maintain web accessibility standards. The Amnesty logo has been added to digest emails.

Text string customisation

Any 'string' on the Discourse platform can be changed to better suit AIUK's community context. Key changes include:

  • Messages → Announcements in Groups (reflects the broadcast nature of group communications)
  • Users Map → Map in the hamburger menu
  • Upcoming Events → Events in the hamburger menu

AI configuration

The Discourse AI plugin has been enabled, using a local hosted 'small' LLM. No external AI services (such as OpenAI) or image generation features are in use — protecting user data from being sent to third-party AI services.

🔍

AI Search

Semantic search helps users more quickly and accurately find what they're looking for across the platform.

🛡️

Spam Detection

Automated spam detection reduces the manual moderation burden on community leaders and staff.

🏷️

Inferred Concepts

If topics such as "self-harm", "hate-crime", or "illegal activity" appear, the system automatically raises a moderation alert even without exact keyword matches.

Chat and trust levels

Chat configuration

Direct messaging is restricted to Trust Level 4 and above to prevent new users having an immediate harassment vector. Chat is enabled for moderators, admins, and Trust Level 4 users, allowing them to discuss moderation decisions privately. All users can reply to chats but cannot initiate them. Both direct messages (usually two users) and chat channels (three or more users) are supported.

Trust level settings

For the pilot, all new users are given default Trust Level 1 because they are invited into the platform. For the public launch, Trust Level 0 should be the default to help AIUK mitigate spam accounts and unused signups.

Anonymous mode

Users can create new topics anonymously via "Enter Anonymous Mode" in the user menu. This enables the posting of sensitive topics while maintaining community safety. Anonymous users cannot reply to existing topics, only create new ones.

Groups and permissions

Groups are collections of users. To help with staff bottlenecking and empower activists, one of the requirements for the Community Platform is decentralised user management.

Public groups

Network groups are openly joinable. The Student Action Network (STAN) is joinable upon request rather than open enrolment.

Closed groups

Closed groups include moderator groups, local groups, and university groups. Users join upon request or are added by admins.

Local and Student groups

For the Pilot, Local Regions as well as Local Groups are being tested. A Local Group leader in Newcastle, for example, would be a member of the North East Region Group and the owner of the Newcastle Group. Student representatives are members of the Student Action Network Group as well as their local university group.

Category moderator groups

Category Moderator Groups are labelled "groupname-cm" and contain the category moderators for a sub-category. For example, the Category Moderator(s) of a local group are also members of a regional group as category moderators, allowing them to moderate both the top-level regional category and their local sub-category.

Group naming conventions

The Group slug must match the category slug (e.g. 'glasgow-west') for the Join/Leave button functionality to work correctly. This naming convention is critical to the platform's permission architecture.

Categories and sub-categories

The Category page is both a navigation hub and a place for topic-based discussion. For example, 'Networks' is a category and 'Queer Rights' is a sub-category. Discussions can happen at both levels.

Get Started categories

  • Onboarding & Training — top-level category for new users
  • All the basics — open sub-category
  • Going further — semi-private, TL2+ only
  • Advanced — semi-private, TL4 and above
  • Socialising & Sharing — completely open category

Find Your Place categories

  • Networks — openly joinable; posts moderated by admins and site-wide moderators
  • Network sub-categories — users must join a network before creating posts; moderated by category moderators
  • STAN — join-on-request; moderated by STAN category moderators
  • Human Rights Education — openly accessible; users must request to join sub-categories
  • Local Regions — openly joinable at top level; local groups require request and approval

Additional features

Calendar and Events

The Discourse Calendar plugin is installed and configured, with community calendars added to all top-level categories. Event creation permissions are granted to admins, moderators, and Trust Level 4 participants.

Locations and Map

The Locations plugin adds map functionality, allowing users to see where community members and groups are located. Location has been switched off for user fields but remains available for topics. Map pins link to 'About' pages for local group categories.

Follow plugin

Users can follow other users and be notified when they post. Statistics about follower counts are deliberately hidden, but notifications are active for when followed users create new topics or reply.

Onboarding flow

A custom onboarding flow developed with Communiteq asks new users for essential information and optional questions to help them get started on the platform, including interests selected from a list of AIUK human rights themes.

Profile configuration

User profiles include biography, social links, and custom user fields. Users can select their areas of human rights interest:

  • Everyday rights
  • Racial justice in the UK
  • Freedom of expression
  • Human rights frameworks
  • Individuals and communities at risk
  • Human rights crises
  • Refugee and migrant rights
  • Gender justice
  • Business and human rights
  • Technology and human rights

Privacy-first profile settings

  • User profiles are hidden from the public
  • Background image uploads restricted to Trust Level 1 and above
  • Full name is mandatory but only visible to admins; not used to suggest usernames
  • Website field restricted to Amnesty domains by default (expandable by admin)
