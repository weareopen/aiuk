Technical Discovery Report
& Recommendations

The primary technical output of the project. This report synthesises research, community insight, platform testing, and strategic analysis into a clear recommendation for AIUK's community platform.

About this report

The Technical Discovery Report and Recommendations is one of the two anchor documents of the AIUK Community Platform project. It is the primary technical output of Phase Five, drawing together everything the project team learned across the preceding phases into a single, actionable recommendation.

The report is the product of four phases of work: community research and consultation (Phase Two), data analysis (Phase Three), community strategy design (Phase Four), and platform analysis and comparison (Phase Five). Everything that preceded it was, in some sense, preparation for the question this report answers: which platform should AIUK use, and how should it be configured?

This report is held in Google Docs and is accessible to project team members.

What the report answers

The report was written to answer the central technical question of the project — not just "which platform?" but the fuller set of questions that a responsible technology recommendation requires:

The core questions

  • Which community platform is the best fit for AIUK's activist community, given their needs, technical context, and organisational constraints?
  • Why that platform — what evidence and reasoning supports the recommendation?
  • What the platform should look like — how should it be structured, governed, and configured for the AIUK context?
  • What risks exist and how should they be managed?
  • What comes next — what does a responsible path to pilot and then to scale look like?

What went into the report

The report draws on a substantial body of research, testing, and strategic work produced across the project. Documents marked ✅ in the project index are those referenced within this report.

Data Report: A little platform, a lot of strategy

✅ Referenced

Community Metrics

✅ Referenced

Processes and Workflows

✅ Referenced

Recognition ecosystem

The proposal for how activist contributions could be recognised and celebrated — a key feature consideration in the platform comparison.

✅ Referenced

Building an AIUK community ecosystem

A map of the broader technology ecosystem the platform would need to fit into — AIUK's website, CRM, knowledge hub, and other tools.

✅ Referenced

Analysis and comparison of technical platforms

A detailed comparison of candidate platforms across a set of criteria derived from the community strategy and user needs.

✅ Referenced

The report also draws on the User Journey, the Audience refinement and personas spectrum, and the user stories developed to represent the range of people who would use the platform.

Structure of the report

The report moves from context and evidence through to analysis and recommendation. It is structured to be read by both technical and non-technical audiences — with enough detail for technical decision-makers and enough narrative for strategists and leadership.

Sections in the report

  • Executive summary — the recommendation and the headline reasoning in brief
  • Project context — what the project set out to do and how the recommendation was reached
  • Community needs — what the research revealed about what AIUK activists need from a platform
  • Platform analysis — how candidate platforms were assessed and compared
  • Recommendation — the recommended platform, with the rationale
  • Configuration guidance — how the platform should be set up for the AIUK context
  • Risks and mitigations — what could go wrong and how to manage it
  • Next steps — the path from recommendation to pilot and beyond

Companion document

The Technical Discovery Report is one of two anchor documents in the project. It should be read alongside the AIUK Final Evaluation Report, which assesses what was achieved during the pilot against the goals set out in this report.

Together, the two reports tell the full story of the project: this report sets out the recommendation and the reasoning; the evaluation report assesses how it played out in practice.
